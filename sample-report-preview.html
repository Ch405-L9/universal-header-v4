FINDINGS
Issues Identified
Render-Blocking Font Requests
Goldman + Satoshi loaded via blocking <link rel="stylesheet">. Delayed FCP on mobile 4G by ~2,020ms.
Est. savings: 2,020ms render delay
Oversized Hero Image — No Responsive Srcset
1600px image served to all viewports including 375px mobile screens. No fetchpriority on LCP element.
Est. savings: 276 KiB on mobile
Third-Party CDN Image Latency
All images served from res.cloudinary.com requiring separate DNS resolution + TLS handshake on every cold load.
Est. savings: 400–800ms on mobile cold load
viewport maximum-scale=1 Blocking Zoom
Pinch-zoom disabled — accessibility violation and Lighthouse Best Practices flag.
Icon Button — No Accessible Name
Hamburger menu button had no aria-label. Screen readers could not identify the control.
Missing OG Meta Tags
og:url, og:type, og:site_name, og:locale absent. Suboptimal link previews on social platforms.
Video Element — No Captions Track
Accessibility best practice requires a <track kind="captions"> element on video.
RESOLVED
Fixes Applied
✓
Async Font Loading Pattern
Converted to media="print" onload swap + <noscript> fallback. Eliminated render-blocking on both font faces.
✓
Hero srcSet at 640w / 1024w / 1600w
Mobile now receives 640px variant (~60KB target). Added fetchPriority="high" to LCP element for browser preload priority.
✓
Viewport maximum-scale Removed
Pinch-zoom restored. Removed Best Practices flag and resolved accessibility violation.
✓
Hamburger aria-label Added
Dynamic aria-label: "Open/Close navigation menu" + aria-expanded + aria-controls wired to mobile-nav panel.
✓
OG Meta Tags Completed
Added og:url, og:type, og:site_name, og:locale, og:image:width/height. All 10 required Open Graph properties present.
✓
JSON-LD Schema @graph Deployed
Organization + LocalBusiness + WebSite + Service entities. Rich Results Test: 0 errors, 0 warnings.
✓
Video Captions Track Added
<track kind="captions"> element added. Accessibility compliance restored.
PATH FORWARD
Recommended Resolution Path — Score Targets: Mobile 95+ · Accessibility 100
01
Migrate Images to Vercel CDN (Local /public/images/)
Eliminate third-party DNS + TLS overhead on every cold load. Serve hero, logo, and report preview as WebP from same origin. Estimated LCP reduction: 400–800ms.
Impact: High · Est. +8–12 LH Points
02
Convert All Images to WebP — Right-Sized Variants
Hero: 640/1024/1600px WebP (target <200KB total). Logo: 400px WebP <15KB. Report preview: 900px WebP <150KB. Favicon SVG minified to <5KB.
Impact: High · Est. +6–10 LH Points
03
Resolve Remaining Contrast Ratio (Accessibility 96 → 100)
BADGRTECH nav subtitle text (zinc-500 on dark bg) falls below WCAG AA 4.5:1 threshold. Elevate to zinc-300 or white/60 opacity without breaking visual hierarchy.
Impact: Medium · Est. +4 Accessibility Points
04
Add WebM Source to Video Element
Serve VP9 WebM alongside MP4. Modern browsers prefer WebM (~30% smaller). Use Cloudinary transform: /vc_vp9,q_auto:good,br_1m/ for on-the-fly transcoding.
Impact: Medium · Video load time –30%
