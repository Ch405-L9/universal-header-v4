1. 30‑day implementation playbook (for junior PM/tech)
Phase 0 – Before Day 1 (sales + contract)

Sign SOW for: (a) after‑hours/weekend AI answering on customer‑owned server, (b) optional 6‑month web‑optimization retainer, (c) optional CRM auto‑ticketing.

Collect initial info: existing phone provider, website platform, email provider, any current CRM/job system.

Week 1 – Discovery, specs, server

Day 1: Discovery call/workshop (3–4 hours): map call flows, booking rules, FAQs, tech routes, and target KPIs.

Day 2: Draft and send 1‑page server spec (CPU/RAM/SSD + Ubuntu LTS + Cockpit); customer orders server.

Day 3–4: On‑site or remote: install Ubuntu Server LTS, harden basics, install Cockpit, set up Docker (or Proxmox + containers if using a hypervisor).

Day 5: Connect dev/test instance to VoIP/PBX test numbers; verify inbound call routing into the AI call handler stub.

Week 2 – AI logic, knowledge base, shadow mode

Day 6: Build/update FAQ + policy doc from discovery; load into KB (RAG store).

Day 7: Configure open‑source LLM (Phi‑class or Mistral‑class) and prompt templates for intake, FAQ answers, and escalation.

Day 8–9: Enable shadow mode: real calls mirrored to AI during business hours; AI produces suggested responses and call summaries (human still answers the phone).

Day 10: Daily review with front‑desk/manager; refine prompts and rules from shadow‑mode output.

Week 3 – Go‑live after hours + CRM base

Day 11–12: Final test calls (internal) for after‑hours flows; verify transcript and log storage; confirm emergency/escalation behavior.

Day 13–14: Go‑live after‑hours/weekend answering; monitor logs in near real time; keep rollback to voicemail ready.

Day 15: Stand up basic micro‑CRM DB and web UI (minimum: customers, tickets, call logs); link AI call handler to create records.

Week 4 – Stabilize + ticket automation + docs

Day 16–17: Implement auto‑ticket creation: when call ends, write structured ticket and notify assigned technician (email/SMS/app).

Day 18–19: Validate ticket quality with techs; adjust fields and assignment rules.

Day 20: Deliver client runbook (how to pause AI, see tickets, request changes); update website and call scripts with privacy notice.

Day 21–22: If retainer signed, baseline web optimization: fix titles/meta, CTAs, NAP, lead forms; set up analytics tracking.

Day 23–30: Light optimization: refine scripts based on real data, confirm KPIs, and prepare first monthly/launch report.
